UniBite — Τι προστέθηκε σε αυτό το push
========================================

Αυτό το push προσθέτει πλούσια δοκιμαστικά δεδομένα (seed) και τα παραδοτέα
(αναφορά, export ΒΔ, screenshots), ώστε η εφαρμογή να φαίνεται γεμάτη και να
καλύπτει όλες τις οθόνες/περιπτώσεις για την παρουσίαση.


1. ΠΛΟΥΣΙΟ SEED  (schema.sql)
----------------------------
Το schema.sql ξαναγράφει τη βάση με ρεαλιστικά demo δεδομένα:

  • 9 χρήστες  — admin + 8 φοιτητές (Αθήνα/ΕΜΠ και Πάτρα), με διαφορετικούς πόντους
  • 18 αγγελίες — Αθήνα (Ζωγράφου) + Πάτρα (Ρίο & κέντρο), ώστε να δουλεύει το
                  φιλτράρισμα κατά απόσταση (maxKm). Καλύπτονται όλες οι καταστάσεις:
                    - Ενεργές (έχουν μερίδες)
                    - Ανενεργές (0 μερίδες — εμφανίζονται γκριζαρισμένες)
                    - Διεγραμμένες (>48h — κρυφές από το feed, μένουν για στατιστικά)
                  Άλλες έχουν δηλωμένα αλλεργιογόνα κι άλλες όχι (για το σχετικό φίλτρο).
  • 20 αιτήματα — σε ΟΛΕΣ τις καταστάσεις: pending, approved, rejected, picked_up, no_show
  • 9 βαθμολογίες — με ποικιλία αστεριών (γεμίζουν avg βαθμολογία + leaderboard).
                    Δύο παραληφθείσες μερίδες μένουν ΧΩΡΙΣ βαθμολογία επίτηδες,
                    ώστε να φαίνεται το widget «αφήστε βαθμολογία».

Έτσι γεμίζουν: το feed (λίστα + χάρτης), ο πίνακας αιτημάτων του μάγειρα
(approve/reject/παραλαβή/μη-παραλαβή), οι κρατήσεις του καταναλωτή με βαθμολόγηση,
και ο πίνακας Διαχειριστή (μηνιαία στατιστικά + Top Donor + κορυφαία γεύματα).

2. ΑΝΑΦΟΡΑ  (report.html)
-------------------------
Αυτόνομο HTML — άνοιγμα σε browser και «Εκτύπωση → Αποθήκευση ως PDF».
Περιέχει: στόχο, τεχνολογίες, σχεδιασμό ΒΔ + ER diagram, περιγραφή πινάκων,
REST API, οδηγίες εκτέλεσης και ενότητα screenshots (Εικ. 1–10).
ΣΗΜ.: συμπληρώστε τα στοιχεία της ομάδας στην κορυφή του αρχείου.


3. EXPORT ΒΑΣΗΣ  (unibite_export.sql)
-------------------------------------
mysqldump της βάσης (σχήμα + δεδομένα) — παραδοτέο μαζί με τον κώδικα.
Αναπαραγωγή:  mysqldump -u root unibite > unibite_export.sql


4. SCREENSHOTS  (screenshots/ + scripts)
----------------------------------------
  screenshots/                 — τα στιγμιότυπα που εμφανίζονται στην αναφορά
  capture-screenshots.js       — auto-capture των desktop οθονών (01–08) με Playwright
  embed-screenshots.js         — ενσωματώνει τα PNG inline στο report.html (self-contained)

Τα 09–10 (mobile/tablet) τραβιούνται χειροκίνητα (DevTools → device mode).

Παραγωγή screenshots (μία φορά εγκατάσταση Playwright):
  npm i -D playwright
  npx playwright install chromium
  npm start                    (σε άλλο terminal)
  node capture-screenshots.js


ΕΚΤΕΛΕΣΗ ΕΦΑΡΜΟΓΗΣ
------------------
  1. mysql -u root < schema.sql        (ΠΡΟΣΟΧΗ: κάνει DROP/CREATE τη βάση unibite)
  2. ρυθμίσεις στο .env (DB_HOST, DB_USER, DB_PASSWORD, DB_NAME)
  3. npm install
  4. npm start   →  http://localhost:3000


ΣΗΜΕΙΩΣΕΙΣ
----------
  • Το schema.sql κάνει DROP DATABASE unibite — μην το τρέξετε σε βάση με πραγματικά δεδομένα.
  • Η Playwright είναι devDependency, χρειάζεται μόνο για την παραγωγή των screenshots.
  • Τα στατιστικά «τελευταίου μήνα» βασίζονται σε picked_up αιτήματα — το seed τα τοποθετεί
    μέσα στο διάστημα, οπότε ο πίνακας Διαχειριστή δεν είναι ποτέ άδειος.
